# КУРСОР — Школа программирования (v2.0, с настоящим бэкендом)

Полная учебная платформа для детской онлайн-школы программирования. Ученики занимаются дома, учителя в реальном времени видят их прогресс, администратор создаёт аккаунты и управляет контентом.

## 🚀 Что нового в v2.0

| Было (v1) | Стало (v2) |
|---|---|
| 100% фронтенд + `localStorage` | **Node.js + Express + SQLite** |
| Открытые пароли в коде, демо-аккаунты в форме входа | **bcrypt-хеши + JWT, форма входа без подсказок** |
| Регистрация и редактирование — у любого | **Только `admin` создаёт аккаунты** |
| Учитель видел `localStorage` своего браузера | **Учитель видит прогресс учеников в реальном времени (WebSocket)** |
| Прогресс терялся при чистке кеша | **Прогресс на сервере, синхронизируется между устройствами** |

## 🧱 Стек

- **Backend:** Node.js 18+, Express 4, better-sqlite3, JWT (`jsonwebtoken`), bcryptjs, `ws`
- **Frontend:** оригинальный HTML/CSS/JS, без сборки и без фреймворков
- **БД:** SQLite-файл в `server/db/kursor.sqlite` (создаётся автоматически)
- **Real-time:** нативные WebSocket-соединения (`/ws?token=…`)

## ⚡ Запуск

```bash
# 1. Установить зависимости
npm install

# 2. (опционально) скопировать .env
cp .env.example .env
# и поменять JWT_SECRET, SEED_ADMIN_LOGIN/PASSWORD

# 3. Запустить сервер (БД создастся и засеется автоматически)
npm start

# Откройте: http://localhost:3000
```

При первом запуске будет создан **один администратор** с логином/паролем из `.env` (по умолчанию `admin / admin`). Модули и задачи из `public/data/database.js` автоматически перенесутся в SQLite — никакого ручного импорта не требуется.

> ⚠️ Сразу после первого входа смените пароль администратора в Профиле или через админ-панель.

## 👥 Роли

| Роль | Может |
|---|---|
| `admin` | Всё: создавать/удалять/редактировать любых пользователей, модули, задачи. Видит прогресс всех учеников. |
| `teacher` | Видит прогресс прикреплённых учеников (или всех, если у них не назначен teacher_id), редактирует модули и задачи. Не может создавать аккаунты. |
| `student` | Решает задачи, видит свой прогресс, рейтинг, профиль. |

Ученика к учителю можно прикрепить через админ-панель (поле «Прикреплённый учитель» в форме редактирования ученика). Если поле пустое — этого ученика видят все учителя.

## 🔐 Безопасность

- Пароли хранятся как **bcrypt-хеши** (cost factor 10), а не в открытом виде.
- Аутентификация — **JWT** (срок жизни задаётся в `.env`, по умолчанию 7 дней).
- Все API-маршруты, кроме `/api/auth/login` и `/api/health`, требуют валидный токен.
- Роли проверяются на сервере, а не только на фронтенде.
- Очки за задачи начисляются **сервером**, фронтенд не может накрутить.
- WebSocket аутентифицируется тем же JWT в querystring (`/ws?token=...`).

## 🌐 REST API (кратко)

| Метод | URL | Доступ | Описание |
|---|---|---|---|
| GET | `/api/health` | публично | Health-check |
| POST | `/api/auth/login` | публично | Логин → `{ token, user }` |
| GET | `/api/auth/me` | auth | Текущий пользователь |
| POST | `/api/auth/change-password` | auth | Сменить свой пароль |
| GET | `/api/users` | admin/teacher | Все пользователи |
| GET | `/api/users/students` | admin/teacher | Только ученики (для учителя — только его) |
| POST | `/api/users` | **admin** | Создать аккаунт |
| PUT | `/api/users/:id` | admin или сам пользователь | Изменить |
| DELETE | `/api/users/:id` | **admin** | Удалить |
| GET | `/api/modules` / `/api/tasks` | auth | Список |
| POST/PUT/DELETE | `/api/modules`, `/api/tasks` | admin/teacher | CRUD контента |
| GET | `/api/progress/me` | student | Мой прогресс |
| GET | `/api/progress` | admin/teacher | Прогресс всех учеников |
| GET | `/api/progress/leaderboard` | auth | Публичный рейтинг |
| POST | `/api/progress/attempt` | student | Записать попытку |
| POST | `/api/progress/complete` | student | Записать решение |

## 📡 WebSocket

Любой `teacher` или `admin`, открывший страницу `pages/teacher.html`, автоматически подключается к `ws://host/ws?token=JWT`. При любом изменении прогресса ученика (попытка / завершение задачи) сервер шлёт всем заинтересованным наблюдателям:

```json
{ "type": "progress", "studentId": "u_…", "progress": { ... }, "t": 1700000000000 }
```

Соответствующая строка в таблице учителя обновляется без перезагрузки и кратко подсвечивается жёлтым.

## 📂 Структура

```
kursor-platform/
├── package.json
├── .env.example
├── README.md
├── server/
│   ├── index.js            ← главный файл (Express + WS)
│   ├── db.js               ← SQLite + схема
│   ├── init-db.js          ← засев администратора и контента
│   ├── auth.js             ← JWT, bcrypt, middleware ролей
│   ├── ws.js               ← WebSocket-сервер
│   ├── routes-auth.js      ← /api/auth/*
│   ├── routes-users.js     ← /api/users/*
│   ├── routes-content.js   ← /api/modules, /api/tasks
│   ├── routes-progress.js  ← /api/progress/*
│   └── db/kursor.sqlite    ← создаётся автоматически
└── public/
    ├── index.html          ← страница входа (без демо-аккаунтов!)
    ├── css/style.css
    ├── data/database.js    ← статические LANGS/GROUPS/MASCOT/BADGES (UI-константы)
    ├── js/
    │   ├── api.js          ← API-клиент + WS-клиент
    │   └── app.js          ← навбар, утилиты
    ├── pages/
    │   ├── dashboard.html  ← главная ученика
    │   ├── catalog.html
    │   ├── task.html       ← решение задачи (с Pyodide для type=code)
    │   ├── profile.html    ← + смена пароля
    │   ├── leaderboard.html
    │   └── teacher.html    ← real-time прогресс
    └── admin/index.html    ← полная админ-панель
```

## 🛠 Полезные команды

```bash
npm start           # запустить сервер (порт 3000 или PORT из .env)
npm run dev         # запустить с авто-перезагрузкой при изменении файлов
npm run init-db     # вручную пере-инициализировать БД
```

## 🔧 Сброс данных

Удалите файл `server/db/kursor.sqlite` (и `.sqlite-wal` / `.sqlite-shm` если есть) — при следующем запуске БД пересоздастся, администратор и весь контент засеются заново.
